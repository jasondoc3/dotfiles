The linear MCP server is a pre-requisite to using this skill. It must be available. If it's not bail, and tell me why.

Start working on the linear issue or issues given with the skill. Before starting work on the issue, run the /linear-grok skill.

Follow these steps once you've groked the issue.

1. Create a new git feature branch from master (or main) before starting implementation. If there's more than one issue, work on them serially in the order that makes the most sense.
2. Once implementation has completed, commit the changes to the feature branch and create a draft PR using the /pr skill
3. I may have feedback once the PR has been created, so continue to work with me until I'm satisfied with the outcome. Once I'm satisfied, we can move onto the next issue if there is one. The PR doesn't need to be merged to move on as it's waiting for human review. Once concluded, we're done!

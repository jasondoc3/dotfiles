Take some of my philosophy for development when coding and building systems by always incorporating the following:
- Attempt to follow Gall's law, which states "A complex system that works is invariably found to have evolved from a simple system that worked. A complex system designed from scratch never works and cannot be patched up to make it work. You have to start over with a working simple system."
- Consider YAGNI when engineering.
- Don't prematurely optimize and avoid fluff.
- If it's a lot of code, question if it's needed.
- If you're unsure if a feature exists within a particular framework or codebase, try referencing documentation.
- Be brief and concise in your responses.

- use the pr skill

When executing commands:
